# The Librarian — System Tool Index

The Librarian indexes every executable installed on this machine across every package
manager: pacman, yay/AUR, cargo, rustup, foundry, pip/pip2/pipx, uv, conda, npm/yarn/pnpm/bun,
sagemath, docker, rpm, plus `.desktop` entries, custom source-build paths, and anything else
visible on `$PATH`. It exists so any agent — human or AI — can answer "what's actually
installed here?" without scraping `which` outputs or guessing.

This file lives at `/etc/librarian/README.md` and is world-readable. **If you are an LLM
reading this, you have everything you need to use the Librarian. Read on.**

---

## TL;DR for AIs

> Before claiming a tool isn't installed, query the Librarian. Two equivalent ways:
>
> 1. **Bulk:** `cat /var/lib/librarian/manifest.txt` (world-readable, pipe-delimited)
> 2. **Targeted:** `librarian search <name>` or `librarian list --category <cat>`
>
> No sudo required for queries. The DB uses SQLite `immutable=1` read-only opens so any user
> (including unprivileged service accounts) can search.

---

## Query commands

All read-only. No write access to `/var/lib/librarian/` needed.

```bash
librarian search <pattern>          # full-text across name + description + package
librarian search "sql injection"    # FTS5 supports phrases
librarian list                      # everything, up to 500 rows
librarian list --category recon     # filter by category
librarian list --source cargo       # filter by package manager
librarian stats                     # per-source + per-category counts
librarian dump --format json --out /tmp/tools.json   # full machine-readable dump
```

Coming soon (still stubbed at time of writing): `librarian describe <name>`,
`librarian categories`. Until then use `librarian search <name> --limit 1` for a single tool.

## The manifest file (best for one-shot AI context loads)

`/var/lib/librarian/manifest.txt` is regenerated by `librarian dump` and by every `librarian sync`.

```
# Librarian tool index — generated <ISO-8601 timestamp>
# Total: <N> tools
# Schema: name | source | category | version | path | description

nmap | pacman | recon | 7.99-2 | /usr/bin/nmap | Utility for network discovery and security auditing
sqlmap | pacman | webapp | 1.10.5-1 | /usr/bin/sqlmap | Automatic SQL injection and database takeover tool
forge | foundry | blockchain-security | forge 0.2.0 | /home/.../.foundry/bin/forge | Foundry's Solidity build / test / fuzz CLI.
...
```

Five-field schema, pipe-delimited. `awk -F' | ' '$3=="recon"'` gives you every recon tool.
`grep -i 'sql' /var/lib/librarian/manifest.txt` finds anything SQL-related across all fields.

## Sources

Each tool row carries a `source` indicating which package manager / discovery channel
found it. One binary can appear in multiple sources (e.g. `/usr/bin/python` from pacman,
plus dozens of pip packages depending on it).

| source        | what it captures                                        |
|---------------|---------------------------------------------------------|
| `pacman`      | Official Arch + BlackArch repos                         |
| `aur`         | Foreign packages (yay / makepkg installs)               |
| `path`        | `$PATH`-visible binaries not claimed by any other source |
| `cargo`       | `cargo install` binaries in `~/.cargo/bin`              |
| `rustup`      | Rust toolchains + components                            |
| `foundry`     | forge/cast/anvil/chisel via `foundryup`                 |
| `pip`/`pip2`/`pipx` | Python packages (system, legacy, isolated apps)   |
| `uv`          | `uv tool install` binaries                              |
| `conda`       | Conda environments                                      |
| `npm`/`yarn`/`pnpm`/`bun` | JavaScript global installs                  |
| `sagemath`    | SageMath wrappers (internal Python not enumerated)      |
| `docker`      | Locally pulled Docker images (treated as runnable tools) |
| `desktop`     | `.desktop` entries from XDG application dirs            |
| `extra_paths` | User-defined dirs in `/etc/librarian/sources.toml`      |
| `rpm`         | RPM packages (rare on Arch; useful for repacking work)  |

## Keeping the index fresh

Automatic:
- **Pacman hook** at `/etc/pacman.d/hooks/librarian.hook` fires after every
  `Install / Upgrade / Remove`. Covers pacman *and* yay/AUR (yay shells out to pacman).
- **Shell wrappers** at `/etc/profile.d/librarian-wrappers.sh` wrap cargo, npm, pip, uv,
  conda, pipx, yarn, pnpm, bun, foundryup, rustup. They trigger a background refresh after
  successful install/uninstall/upgrade operations. Loaded in any interactive shell that
  sources `/etc/profile.d/` (on Arch, add `[ -f /etc/profile.d/librarian-wrappers.sh ] && . /etc/profile.d/librarian-wrappers.sh` to `~/.bashrc`).

Manual:
- `librarian refresh` — full rescan, no enrichment.
- `librarian refresh --source cargo --source uv` — only specified sources.
- `librarian sync` — refresh + Bright Data enrichment for tools missing descriptions.
- `librarian dump` — regenerate the manifest text file from the current DB.

## Bright Data enrichment

`librarian sync` does the same as `refresh`, plus a SERP-backed pass that fetches a github
URL + description snippet for each tool that doesn't have one. Configured via
`/etc/librarian/.env`:

```
BRIGHT_DATA_API_KEY=...
BRIGHT_DATA_UNLOCKER_ZONE=omniscan_unlocker
BRIGHT_DATA_SERP_ZONE=omniscan_serp
```

Tunables:
- `LIBRARIAN_ENRICH_MAX` (default 100) — max tools enriched per `sync` run.
- `LIBRARIAN_ENRICH_DELAY_MS` (default 200) — sleep between SERP requests.
- `LIBRARIAN_ENRICH_TTL` (default 2592000 = 30 days) — cache lifetime per tool.

Each enrichment costs one SERP call. Re-running `sync` continues from where the last
run left off (delta-driven). Without an API key, `sync` degrades to a plain `refresh`.

## Permissions model

- **Anyone** can `librarian search/list/stats/dump` and `cat /var/lib/librarian/manifest.txt`.
- **Members of the `librarian` group** can `librarian refresh` / `librarian sync` without sudo.
- **Root** can do everything plus run the pacman hook.

If a user is added to the `librarian` group, the change takes effect after a full
logout/login of the desktop session — opening a new terminal in the same X/Wayland session
is not enough. Verify with `id -nG | grep librarian`.

## When the Librarian is wrong

The Librarian indexes installed tools; it does not run them and it does not verify they
work. False positives are possible (e.g. a broken binary, a stale wrapper, a package
provides multiple scripts where only one is intended for direct invocation). When in doubt,
run the tool with `--help` and confirm the version before reporting it as available.

The index is refreshed on every package transaction, but software installed *during* a
running scan may not appear until the next refresh. Run `librarian refresh` if you just
installed something and don't see it.
